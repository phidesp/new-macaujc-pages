# new-macaujc-pages

## 项目简介

本仓库用于归档和发布多个独立 HTML 页面。这些页面均为静态内容，不依赖于特定域名或外部服务器，可直接通过浏览器或 GitHub Pages 等服务访问。

仓库的主要目的是集中管理和保存一系列 HTML 文件，便于后续查阅、分享或进一步开发。

## 目录说明

```
/
├── index.html          # 仓库入口页面（可选）
├── pages/              # 主要 HTML 页面存放目录
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/             # 静态资源目录（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：存放所有 HTML 页面文件，每个文件对应一个独立页面。
- `assets/`：存放页面所需的样式、脚本、图片等资源文件，按类型分目录管理。

## 页面归档说明

所有 HTML 页面均以独立文件形式存储，页面之间无强依赖关系。每个页面可单独打开或部署。

- 页面内容可能包含静态信息展示、交互功能或实验性代码。
- 各页面功能、样式、用途可能不同，具体请参考页面内说明。
- 页面文件命名应简洁明了，便于识别。

## 维护说明

- 本仓库欢迎提交新页面或改进现有页面。请确保提交的 HTML 文件结构完整、无外部不可用资源。
- 提交新页面时，建议在 `pages/` 目录下添加，并更新 `index.html` 或本 README 的导航部分（如适用）。
- 请勿在页面中包含任何恶意代码、广告内容或违反开源社区规范的信息。
- 仓库维护者会定期检查提交内容，合并合理请求，并清理无效或过时页面。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接打开 `pages/` 目录下的 HTML 文件，即可在浏览器中查看。
3. 若启用 GitHub Pages，可设置发布源为根目录或 `docs/` 目录，通过域名访问。
